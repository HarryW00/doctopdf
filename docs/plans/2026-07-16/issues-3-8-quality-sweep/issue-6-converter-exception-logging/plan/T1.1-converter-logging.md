# Worker Prompt: T1.1-converter-logging

- **Source task**: Batch 1 / Task 1 — Add stdlib logging to Word cleanup/quit/restart helpers in `converter.py`

---

## 1. Mission & Rules

### Mission

Add a module-level stdlib logger to `doctopdf/converter.py` and replace every silent `except Exception: pass` in the Word cleanup/quit/process-check helpers with a `debug` log, plus a `warning` when `restart_word` fails to relaunch Word — without changing best-effort (non-raising) behaviour.

### Rules

- Follow the Scope in Section 5 — modify ONLY `doctopdf/converter.py`.
- Do NOT call `logging.basicConfig` (that is the CLI's job, owned by worker T1.2).
- If a target line/function cannot be found, report it — do not guess.
- Do not add new dependencies (stdlib `logging` only).
- Workers are leaf nodes — do not spawn sub-workers.

---

## 2. Context

### Input Files

- `doctopdf/converter.py` — the only file to edit. Contains `WordConverter` with staticmethods `_cleanup_stale_word`, `quit_word`, `_force_quit_word`, `_is_word_running`, `restart_word`.
- `references/python-logging.md` — the exact logging pattern (module `_log = logging.getLogger(__name__)`; emit with `%`-style args; never `basicConfig` in library code).

### Background

These helpers currently swallow all exceptions with bare `except Exception: pass`, making intermittent cleanup failures invisible (SPEC #6, Req 1–3). The fix keeps them best-effort (still never raise) but records failures. `restart_word` currently returns silently when Word fails to relaunch, causing a cryptic failure on the next conversion — it must now emit a `warning`.

1. Locate `restart_word`. It ends with this loop and a trailing comment:

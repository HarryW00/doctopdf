## 3. Tasks

### `doctopdf/cli.py` — add `import logging`

1. Open `doctopdf/cli.py`.
2. In the top import block, add `import logging` after `import argparse`:
   - **Before**:
     ```python
     import argparse
     import sys
     ```
   - **After**:
     ```python
     import argparse
     import logging
     import sys
     ```

### `doctopdf/cli.py` — add `--verbose` flag to `build_parser()`

1. In `build_parser()`, in the "Utility" argument group (near `--check`), add the flag before the `--check` argument:
   ```python
       parser.add_argument(
           '--verbose',
           action='store_true',
           help='Enable verbose debug logging (shows internal Word cleanup/restart diagnostics)',
       )
   ```

### `doctopdf/cli.py` — add `_configure_logging` helper

1. Add this function after `build_parser()` (after its `return parser`) and before `cmd_check()`:
   ```python


   def _configure_logging(verbose: bool) -> None:
       """Configure stdlib logging: warnings visible by default, debug with --verbose."""
       logging.basicConfig(
           level=logging.DEBUG if verbose else logging.WARNING,
           format='%(levelname)s %(name)s: %(message)s',
           stream=sys.stderr,
       )
   ```

### `doctopdf/cli.py` — wire it into `main()`

1. In `main()`, immediately after `args = parser.parse_args(argv)` and before the mode-dispatch comment (`# ── Mode dispatch`), add:
   ```python
       # Configure stdlib logging before any work; warnings show by default,
       # debug diagnostics require --verbose.
       _configure_logging(args.verbose)
   ```

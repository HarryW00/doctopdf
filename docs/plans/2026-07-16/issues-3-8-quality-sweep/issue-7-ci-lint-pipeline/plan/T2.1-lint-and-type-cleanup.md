---

## 1. Mission & Rules

### Mission

Run `ruff check doctopdf/` and `mypy doctopdf/ --ignore-missing-imports` (using the config added by T1.2) and fix every violation in `doctopdf/` until both exit 0, so CI is green on its first run.

### Rules
- Modify ONLY files under `doctopdf/` (the package). Do NOT touch `tests/`, `pyproject.toml`, `README.md`, or `.github/`.
- **Behavior-preserving**: fixes are limited to style/formatting, unused-import removal, adding type annotations, and similar non-semantic changes. Do not alter runtime logic.
- Prefer `ruff check --fix doctopdf/` for safe auto-fixes; handle the rest manually.
- Report EVERY file you change and a one-line reason for each.
- Depends on Batch 1 (T1.2's ruff/mypy config must be in place before running).
- Workers are leaf nodes.

## 3. Tasks

1. From the repo root, ensure tooling is installed: `pip install -e ".[dev]"`.
2. Run `ruff check doctopdf/`. For each violation:
   - Apply `ruff check --fix doctopdf/` for safe auto-fixes; re-run to confirm.
   - Resolve any remaining violations manually (formatting to line-length 100, removing unused imports, etc.).
3. Run `mypy doctopdf/ --ignore-missing-imports`. For each type error:
   - Add/fix type annotations or correct the typing; do not change runtime behaviour. (If a reported error is a genuine third-party-stub gap already covered by `ignore_missing_imports`, note it and leave it.)
4. Iterate until both commands exit 0.

### Output
Report: **Files modified** (each `doctopdf/*.py` touched + one-line reason), the final `ruff` and `mypy` exit status, **risks or concerns** (especially any change that was not purely cosmetic).
